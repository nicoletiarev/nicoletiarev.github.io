[NOMBRE DEL PROYECTO: Dashboard de Ventas Regionales]

**Tecnologías:** Power BI, SQL, Python (Pandas) **Foco:** Business Intelligence, Modelado de Datos, Visualización.

Resumen y Pregunta de Negocio

El proyecto abordó la necesidad crítica de la gerencia de ventas para tener visibilidad del rendimiento en tiempo real. Se respondió a la pregunta: **"¿Qué combinación de producto y región está generando la mayor pérdida de margen de beneficio?"**

1. Proceso y Metodología

Se implementó un ciclo de análisis iterativo. El proceso de **ETL** (Extracción, Transformación, Carga) se realizó mediante scripts SQL para depurar y optimizar datos provenientes de múltiples fuentes, garantizando la integridad de los datos en el modelo Estrella.

Diagrama del Modelo de Datos optimizado en Power BI.

2. Hallazgos Clave y Recomendaciones

  • **Hallazgo #1 (Impacto en Margen):** El 60% de las ventas en el segmento C generaron solo el 15% del margen total.
  • **Recomendación:** Se propuso reducir la fuerza de ventas dedicada al segmento C y reenfocar recursos al segmento A, con un potencial de crecimiento del margen del 20%.
